# 拍照机器人项目会议纪要整理
## 一、项目基本定位
本项目计划基于银河通用 G1 轮式双臂机器人，结合灵心巧手灵巧手、手机夹具、影石 Insta360 相机、口袋打印机、补光灯等外设，开发一个面向会场、毕业典礼、展会、活动现场等场景的智能拍照机器人 / 具身摄影助理。
项目目标不是简单地让机器人“按一下快门”，而是希望机器人能够完成一整套拍照服务流程，包括：
1. 在场地中巡航或待命；
2. 发现或接待有拍照需求的用户；
3. 引导用户到合适的拍摄机位；
4. 根据用户选择，使用用户手机或机器人自带相机拍摄；
5. 通过语音引导用户站位、看镜头、摆姿势；
6. 机器人自动调整机械臂和相机位置，完成构图；
7. 触发拍照或录制短视频；
8. 可选打印照片；
9. 可选提供电子照片下载。
整体上，这是一个面向活动场景的半解决方案式机器人摄影服务系统。

## 二、硬件方案
### 1. 机器人本体
项目基于银河通用的轮式双臂机器人平台。
机器人承担的主要任务包括：
- 场地内移动；
- 定点导航；
- 到达预设拍摄机位；
- 接近用户；
- 与用户交互；
- 控制机械臂姿态；
- 承载相机、手机夹具、打印机、补光灯等外设。
### 2. 左臂：工具臂改装
左臂主要作为“摄影工具臂”。
当前设想是在左手腕末端进行结构改装，增加两个工具：
#### 工具一：手机支架 / 手机夹具
在左手腕正前方安装一个带弹簧卡扣的手机支架。
结构方式：
- 通过 3D 打印设计转接件；
- 转接件通过螺丝固定到机器人手腕法兰上；
- 用户将手机放入夹具；
- 机器人通过夹具稳定持握用户手机；
- 手机打开相机界面后，机器人可以利用手机完成拍摄。
这个方案的核心价值是：
> 照片直接保存在用户自己的手机里，隐私性更好，也不需要额外传输电子照片。
#### 工具二：Insta360 Link 2C 相机
在左手腕侧面，即与正前方夹具大约 90° 垂直的位置，固定一台影石 Insta360 Link 2C 相机。
这个相机用于机器人自带拍摄方案。
特点是：
- 可以通过 USB 接入机器人主机；
- 机器人可以直接读取相机画面；
- 可以做更完整的视觉分析；
- 可以做构图、曝光、光线、人脸、阴影等检测；
- 后续可以拓展为照片评分、美图、短视频拍摄等功能。
### 3. 右臂：灵巧手
右手安装灵心巧手的灵巧手，主要承担交互和精细操作。
当前设想中，右手主要执行：
- 伸出食指；
- 其他手指闭合；
- 点击手机屏幕上的拍照按钮；
- 从口袋打印机中抽取打印完成的照片；
- 将照片递给用户；
- 也可以做简单的手势交互。
其中最关键的动作是：
> 机器人左手固定手机，右手灵巧手点击手机相机界面的白色快门按钮。
### 4. 口袋打印机
计划在左臂大臂或机器人机身附近固定一个口袋打印机。
该打印机类似“不带镜头的拍立得”，用于输出小尺寸实体照片。
流程设想：
1. 机器人使用自带 Insta360 相机拍摄；
2. Jetson 或机器人主机处理照片；
3. 通过无线或有线方式发送给口袋打印机；
4. 打印机输出照片；
5. 右手灵巧手抽出照片；
6. 机器人将照片递给用户。
这个功能可以作为现场互动亮点，也可以作为商业化付费点。
### 5. 补光灯
计划在右肩或机器人上半身增加一个补光灯。
设想是使用偏氛围感的幽蓝紫色补光灯，增强现场视觉效果，也让机器人更像一个“摄影助理”而不是普通机器人。
补光灯的作用包括：
- 改善暗光环境下的人像效果；
- 增强仪式感；
- 提升 demo 观感；
- 让机器人在会场中更吸引人。
### 6. 外接显示屏
目前存在一个重要缺口：电子照片如何交付给用户。
如果使用用户手机拍照，照片直接保存在用户手机里，不需要传输。
但如果使用 Insta360 相机拍照，除了打印实体照片外，还需要提供电子版照片。
理想方案是：
- 机器人机身显示二维码；
- 用户扫码下载照片。
但目前银河通用机器人胸前自带显示屏接口可能暂时拿不到，无法直接显示内容。
因此可能需要额外固定一个显示屏在机器人机身上，用于：
- 展示二维码；
- 展示拍照状态；
- 展示付款码；
- 展示照片预览；
- 展示服务菜单。

## 三、两种拍摄方案
项目中有两条拍摄路线。
## 方案 A：使用用户手机拍照
这是隐私性更好的方案。
流程如下：
1. 用户打开手机相机界面；
2. 用户将手机放入机器人左手腕的手机夹具；
3. 机器人通过头部摄像头观察手机屏幕画面；
4. 机器人根据画面进行构图判断；
5. 机器人移动左臂，调整手机角度；
6. 机器人通过语音提示用户站位；
7. 右手灵巧手伸出食指；
8. 点击手机屏幕上的快门按钮；
9. 照片直接保存在用户手机中；
10. 机器人归还手机。
这个方案的优势：
- 用户隐私更好；
- 照片不经过机器人服务器；
- 不需要做照片传输；
- 用户体验直观；
- 很适合现场 demo。
这个方案的不足：
- 机器人只能通过头部摄像头“看手机屏幕”，画面质量受限；
- 手机屏幕反光、亮度、角度可能影响识别；
- 不同手机相机界面可能略有差异；
- 灵巧手点击屏幕需要足够稳定；
- 手机夹具需要适配不同手机尺寸。
## 方案 B：使用机器人自带 Insta360 相机拍照
这是开发空间更大的方案。
流程如下：
1. 机器人使用左腕侧面的 Insta360 Link 2C 相机；
2. 通过 USB 将画面接入机器人主机；
3. 主机直接读取相机画面；
4. 系统进行构图、人脸、曝光、光线、阴影等分析；
5. 机器人语音引导用户；
6. 机器人拍摄照片或短视频；
7. 照片可以打印；
8. 电子照片可以通过二维码传输给用户。
这个方案的优势：
- 机器人直接拿到原始画面；
- 视觉算法开发空间更大；
- 可以做构图评分；
- 可以做曝光检测；
- 可以做阴阳脸检测；
- 可以做人脸数量检测；
- 可以做最佳照片筛选；
- 可以训练或微调小模型；
- 可以拓展智能美图、短视频、活动传播等功能。
这个方案的不足：
- 需要解决电子照片交付；
- 涉及用户肖像隐私；
- 需要设计数据存储和删除机制；
- 需要额外显示屏或扫码下载方案；
- 需要接入 Insta360 SDK 或摄像头数据流。

## 四、用户交互流程
项目希望机器人不是被动设备，而是能像活动现场的摄影助理一样主动服务。
### 1. 巡航与发现用户
在没有任务时，机器人可以在活动场地中巡航。
机器人可以观察是否有人：
- 接近机器人；
- 驻足观看；
- 对机器人表现出兴趣；
- 做出拍照手势；
- 扫码发起请求；
- 通过语音呼叫机器人。
当机器人判断用户可能有拍照需求时，可以主动询问：
> 你好，需要我帮你拍照吗？
### 2. 引导用户到合适机位
如果用户需要拍照，机器人可以：
- 带用户前往预设拍摄点；
- 或跟随用户到用户指定地点；
- 或根据场景判断一个合适角度。
当前比较现实的方式是：
- 在场地中预设若干最佳机位；
- 在导航地图中打点；
- 机器人将用户带到这些位置；
- 或机器人自己移动到对应拍摄点。
### 3. 选择拍摄模式
机器人可以询问用户：
> 你想用自己的手机拍，还是用我的相机拍？
可选模式包括：
- 用户手机拍照；
- 机器人相机拍照；
- 打印照片；
- 拍摄短视频；
- 拍照 + 打印；
- 拍照 + 电子版下载。
### 4. 拍照过程中的语音引导
机器人在拍照过程中可以发出提示：
- “请看镜头。”
- “三、二、一，茄子。”
- “左边的同学稍微靠右一点。”
- “后排靠近一点。”
- “请站到中间。”
- “稍微抬头。”
- “我们再来一张。”
- “这张不错，我帮你们打印出来。”
### 5. 短视频模式
除了拍照，还可以做一个 5 秒左右的短视频模式。
例如：
1. 机器人说：“我们来录一个 5 秒短视频。”
2. “所有人听我口令，一起喊一句口号。”
3. “准备，五、四、三、二、一。”
4. 大家一起喊：“毕业快乐！”或活动口号。
5. 机器人完成视频录制。
这个功能适合活动现场，尤其适合品牌传播和会场互动。

## 五、软件系统与技术架构
项目不能只依靠底层控制模型完成，因为涉及多个设备、多个模式和多个任务分支。
因此需要在执行层之上搭建一个应用层调度系统 / 智能体大脑。
## 1. 执行层
执行层包括：
- 底盘移动；
- 机械臂控制；
- 灵巧手控制；
- 头部摄像头感知；
- Insta360 相机画面读取；
- 手机屏幕识别；
- 打印机连接；
- 补光灯控制；
- 二维码显示；
- 语音播放；
- 语音识别。
这些是具体设备和传感器的调用。
## 2. 感知层
感知层包括：
- 人体检测；
- 人脸检测；
- 手机屏幕识别；
- 拍照按钮定位；
- 构图判断；
- 曝光判断；
- 阴影判断；
- 人物是否入镜；
- 人物是否居中；
- 是否有人闭眼；
- 是否模糊；
- 是否需要补光。
## 3. 应用调度层
应用调度层负责把所有模块串起来。
它需要决定：
- 当前是否有用户需要服务；
- 是否进入拍照流程；
- 使用用户手机还是机器人相机；
- 是否需要打印；
- 是否需要短视频；
- 机器人移动到哪个机位；
- 机械臂应该调整到什么姿态；
- 什么时候语音提示；
- 什么时候按快门；
- 拍完之后如何交付照片。
这个调度系统可以设计成智能体形式。
它需要具备：
- 分支逻辑；
- 循环逻辑；
- 状态管理；
- 工具调用；
- 多设备调度；
- 人机交互决策。
可以理解为：
> 给机器人做一个摄影服务大脑。

## 六、导航与场地部署
机器人需要具备基本的定位、建图和导航能力。
当前设想是：
- 使用激光雷达进行建图；
- 进行场地地图构建；
- 在地图中预设最佳拍摄点；
- 机器人可以导航到这些点；
- 用户也可以带机器人到指定位置；
- 机器人也可以跟随用户。
比较现实的黑客松方案是：
1. 提前对场地进行建图；
2. 在地图中标记几个拍摄机位；
3. 机器人只在这些点之间移动；
4. 不追求完全开放场景自主探索。
这样更稳定，也更适合短时间 demo。

## 七、项目技术难点
当前识别出的技术难点主要包括：
### 1. 机械结构改装
包括：
- 手机夹具结构设计；
- 法兰转接件设计；
- Insta360 相机固定结构；
- 口袋打印机固定结构；
- 补光灯固定结构；
- 显示屏固定结构；
- 线缆管理；
- 设备供电；
- 不影响机器人原有运动空间。
### 2. 机械臂与灵巧手控制
包括：
- 左臂稳定持握手机；
- 左臂调整手机角度；
- 右手伸出食指；
- 右手点击手机快门；
- 右手抽取打印照片；
- 右手递交照片给用户；
- 避免碰撞；
- 保证动作足够稳定。
### 3. 手机屏幕识别与点击
如果使用用户手机方案，需要解决：
- 识别手机屏幕；
- 识别相机界面；
- 定位白色快门按钮；
- 判断当前画面构图；
- 控制灵巧手准确点击按钮；
- 适配不同手机尺寸和界面。
### 4. Insta360 相机接入
如果使用机器人自带相机方案，需要解决：
- SDK 接入；
- USB 画面读取；
- 拍照控制；
- 视频录制控制；
- 图像保存；
- 图像传输；
- 与机器人主控系统联动。
### 5. 打印机接入
包括：
- 打印机通信；
- 图片格式转换；
- 打印任务发送；
- 打印状态检测；
- 灵巧手取照片动作；
- 打印失败处理。
### 6. 电子照片交付
当前还缺少明确方案。
可选方案：
- 外接显示屏显示二维码；
- Web 页面上传照片；
- 局域网临时下载链接；
- 小程序/网页扫码；
- 蓝牙传输；
- AirDrop 类似方案；
- 用户输入手机号或邮箱；
- 机器人生成一次性取图码。
最现实的是：
> 外接一块小屏幕，显示二维码，用户扫码下载照片。
### 7. 应用层智能体调度
这是项目从“功能拼接”变成“完整产品”的关键。
难点包括：
- 多设备调用；
- 多模式切换；
- 用户状态判断；
- 拍照流程状态机；
- 错误恢复；
- 语音交互；
- 任务中断与重试；
- 拍照、打印、传图之间的流程衔接。

## 八、隐私与数据处理
项目中有两个隐私等级不同的方案。
### 方案 A：用户手机拍照
隐私优势较强：
- 照片保存在用户手机；
- 机器人不保存照片；
- 机器人只是通过头部摄像头观察手机屏幕；
- 可以声明只做端侧构图建议，不存储用户肖像信息。
适合强调隐私友好。
### 方案 B：机器人相机拍照
需要更明确的数据策略：
- 是否保存原图；
- 保存多久；
- 是否上传服务器；
- 是否只本地处理；
- 用户扫码后是否自动删除；
- 是否用于模型训练；
- 是否经过用户授权。
建议黑客松里明确说：
> 所有照片默认本地处理，用户下载后可自动删除，不进行未经授权的云端存储和模型训练。
这样会更完整。

## 九、商业模式
项目可以同时讲 To C 和 To B。
## 1. To C：面向个人用户的拍照服务
场景类似街头摄影师或景区拍照服务。
用户可以为以下服务付费：
- 拍一张照片；
- 拍一组照片；
- 打印照片；
- 短视频拍摄；
- 精修照片；
- 高清照片下载；
- 活动纪念照。
价格可以是：
- 拍照 3–5 元；
- 打印另收费；
- 精修另收费；
- 套餐收费。
黑客松测试阶段可以免费体验。
## 2. To B：面向活动主办方的机器人摄影师租赁
适用场景：
- 大型会展；
- 毕业典礼；
- 发布会；
- 企业年会；
- 商场活动；
- 景区打卡点；
- 品牌快闪活动；
- 校园开放日。
价值包括：
- 替代部分基础摄影工作；
- 降低人力成本；
- 增加活动科技感；
- 吸引用户互动；
- 生成传播素材；
- 提升活动体验；
- 形成品牌噱头。
商业形式包括：
- 机器人按天租赁；
- 按活动收费；
- 按拍摄次数收费；
- 品牌定制；
- 打印照片收费；
- 照片下载收费；
- 活动数据报告收费。
这个项目不一定要说完全替代专业摄影师，更合理的说法是：
> 在大量重复、标准化、互动型拍照场景中，机器人可以承担基础拍摄和现场互动工作，专业摄影师则保留高端创作和复杂拍摄任务。
---
## 十、对银河通用的产品价值
从银河通用 G1 的生态角度，这个项目可以被包装成一种半解决方案。
银河通用的开发者选项可以理解为三类：
1. 普通 SDK：提供代码接口；
2. 半封装解决方案：面向具体场景的应用模块；
3. 整体解决方案：如无人零售仓等完整产品。
本项目适合被定位为：
> 面向会场与活动场景的机器人摄影半解决方案。
部署时，客户需要做的事情相对有限：
1. 对场地进行激光雷达建图；
2. 在地图中标记最佳拍摄机位；
3. 配置活动话术；
4. 配置拍照模式；
5. 配置打印和照片交付方式；
6. 启动机器人服务。
机器人即可在场地内提供拍照、短视频、打印和互动服务。

## 十一、黑客松需要补齐的内容
目前项目想法已经比较完整，但黑客松方案还需要进一步明确以下内容。
### 1. 明确 MVP
建议不要一开始做全部功能。
最小可行版本可以设定为：
> 机器人在固定区域内接待用户，使用手机夹具或 Insta360 相机完成一次拍照，并能语音引导用户、完成快门触发和照片交付。
MVP 必须包括：
- 一个稳定拍照流程；
- 一个稳定的人机交互流程；
- 一个能展示 G1 + 灵巧手价值的动作；
- 一个清晰的商业场景。
### 2. 明确主拍摄路线
需要决定黑客松主线到底优先做：
- 用户手机拍照；
- 还是 Insta360 相机拍照。
我的建议是：
> 主线用 Insta360 相机，因为算法和展示空间更大；
辅线展示用户手机夹具和灵巧手点击，作为隐私友好模式和机械亮点。
原因是：
- Insta360 画面可控；
- 视觉处理更容易；
- 电子照片可以展示；
- 可以做构图评价；
- 可以做打印；
- 更容易讲 AI。
但手机方案可以作为非常吸睛的“机器人真的帮你按手机快门”的 demo。
### 3. 明确照片交付方案
必须尽快确定：
- 有没有外接屏；
- 二维码在哪里显示；
- 照片存在哪里；
- 用户如何下载；
- 下载后是否删除；
- 如果没有电子交付，是否只做打印。
### 4. 明确是否打印
打印功能很有互动性，但也会增加工程风险。
建议作为加分项：
- 主线先保证拍照成功；
- 打印作为第二阶段功能；
- 如果打印机接入顺利，再加入正式 demo。
### 5. 明确导航复杂度
不要把黑客松成败压在复杂导航上。
建议：
- 预设 2–3 个拍摄机位；
- 机器人只在固定区域移动；
- 避免开放场景自由巡航；
- 巡航可以用伪巡航或半自动方式展示。
### 6. 明确智能体边界
智能体不要一上来做得太大。
可以先实现一个状态机式 agent：
```plaintext
待机
  ↓
发现用户 / 用户扫码
  ↓
询问拍摄需求
  ↓
选择拍摄模式
  ↓
移动到机位
  ↓
构图检测
  ↓
语音引导
  ↓
拍照
  ↓
打印 / 下载
  ↓
归还 / 结束服务
  ↓
回到待机

```
这样黑客松里比较稳。

## 十二、推荐的项目表达
可以把项目介绍整理成下面这段：
> 本项目基于银河通用 G1 轮式双臂机器人与灵心巧手，开发一套面向毕业典礼、展会、景区和大型活动的具身摄影服务系统。机器人能够在场地中巡航或待命，主动接待有拍照需求的用户，并通过预设机位导航、视觉构图分析、语音站位引导、机械臂相机控制、灵巧手触屏操作、照片打印与二维码交付等能力，完成从“发现用户”到“拍摄交付”的完整闭环。项目同时支持用户手机拍照模式和机器人自带相机拍照模式，前者强调隐私友好，后者支持更丰富的构图分析、照片评分、打印和活动传播功能。该系统可作为银河通用机器人面向活动场景的摄影半解决方案，具备 To C 拍照付费和 To B 活动租赁两类商业化路径。
